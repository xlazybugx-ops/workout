# Драконис · Кодекс Чешуи — структура проекта

Игра разделена на логические модули. Файлы подключаются в `index.html`
обычными `<script src>` в строгом порядке — все модули разделяют общую
область видимости, порядок менять нельзя.

## Файлы

| Файл | Содержимое | Когда редактировать |
|---|---|---|
| `index.html` | Разметка экранов и подключение модулей | Новые экраны/секции |
| `css/style.css` | Все стили игры | Любой внешний вид |
| `js/01-data-core.js` | Стихии, 15 видов драконов, визуал (SVG+фото), древо талантов, миры/биомы/портал | Новые виды, миры, таланты |
| `js/02-data-content.js` | Сундуки/ключи/украшения, свитки легенд, сокровищница, боссы, точки полёта, артефакты, константы кузницы | Новые артефакты, свитки, боссы, лут |
| `js/03-state.js` | Объект состояния `S`, квесты, генетика (бюджет), характеры, забота о драконах | Новые поля сохранения, характеры |
| `js/04-systems.js` | Утилиты, типизированные яйца, выдача/дроп артефактов, переработка | Логика дропа и яиц |
| `js/05-ui-base.js` | Полоса ресурсов, карточки драконов, Логово, экипировка, Гнездо | UI логова и гнезда |
| `js/06-world.js` | Сцены биомов, карта странствий, портал, вызов босса, Хаб, украшения, Сокровищница | Экраны мира и хаба |
| `js/07-flight.js` | Полёт-экспедиция по точкам, мини-игра «поймай добычу» | Механика полёта |
| `js/08-battle.js` | Арена, боевой цикл, мана, механики боссов, мини-игры (тайминг, двойная ульта) | Всё про бой |
| `js/09-forge-breed.js` | Визуализация генома, Гнездилище (скрещивание), рендер кузницы, ковка | Кузница и селекция |
| `js/10-spire-codex.js` | Шпиль (таланты, книга), Восхождение, Кодекс (виды+легенды), вкладки | Шпиль и кодекс |
| `js/11-save-init.js` | localStorage, миграции старых сохранений, idle-доход, ежедневная петля, онбординг, запуск | Сохранение и старт |

## Правила работы

1. **Порядок подключения в index.html не менять** — данные (01–02) грузятся
   раньше кода, который их использует при старте (11).
2. **Новое поле в сохранении** → добавить дефолт в `S` (03) **и** миграцию
   в `migrateDragons()` (11).
3. **Проверка после правок**: `node --check js/имя-файла.js` для синтаксиса.
4. Деплой на GitHub Pages — всей папкой, как раньше (index.html в корне).
